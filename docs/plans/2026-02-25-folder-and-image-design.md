# 文件夹和图片功能设计方案

**日期**: 2026-02-25
**状态**: 已批准

## 概述

为笔记应用添加完整的文件夹管理和图片支持功能，参考 macOS 备忘录的交互方式。

## 核心功能

### 1. 文件夹系统
- 笔记归属到文件夹（一对一关系）
- 文件夹支持嵌套（子文件夹）
- 拖拽笔记到文件夹移动
- 右键笔记选择移动到文件夹
- 文件夹右键菜单（重命名、删除、新建子文件夹）
- 智能文件夹功能暂不实现

### 2. 图片支持
- 支持从剪贴板粘贴图片
- 支持拖拽图片文件到编辑器
- 图片以 Base64 存储在笔记内容中
- 自动压缩图片（最大 1920px，JPEG 质量 80%）
- 编辑器中可调整图片大小

## 数据结构设计

### 数据库变更

#### 1. notes 表添加字段
```sql
ALTER TABLE notes ADD COLUMN folder_id TEXT DEFAULT 'all';
CREATE INDEX idx_notes_folder ON notes(folder_id);
```

#### 2. 新建 folders 表
```sql
CREATE TABLE folders (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  parent_id TEXT,  -- NULL 表示根文件夹
  icon TEXT DEFAULT 'folder',
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL,
  sort_order INTEGER DEFAULT 0,  -- 用户自定义排序
  FOREIGN KEY (parent_id) REFERENCES folders(id)
);

CREATE INDEX idx_folders_parent ON folders(parent_id);
```

#### 3. 系统文件夹
- `all` - 所有笔记（虚拟文件夹，不存储在数据库）
- 用户创建的文件夹存储在数据库中

### localStorage 迁移
- 当前文件夹数据在 localStorage，需要迁移到数据库
- 首次启动时检查并迁移现有文件夹

## 文件夹功能实现

### 数据库层 (electron/db/folders.ts)

**核心函数：**
- `getAllFolders()` - 获取所有文件夹（树形结构）
- `createFolder(name, parent_id)` - 创建文件夹/子文件夹
- `updateFolder(id, changes)` - 重命名文件夹
- `deleteFolder(id)` - 删除文件夹
- `moveFolderToParent(id, new_parent_id)` - 移动文件夹位置

### 笔记关联 (electron/db/notes.ts)

**新增/修改函数：**
- `createNote()` 添加 `folder_id` 参数
- `moveNoteToFolder(note_id, folder_id)` - 移动笔记到文件夹
- `getNotesByFolder(folder_id)` - 获取文件夹下的笔记
- `getNotesCountByFolder(folder_id)` - 获取文件夹笔记数量（包括子文件夹）

### 删除文件夹处理

当删除包含笔记或子文件夹的文件夹时，弹出对话框让用户选择：
1. 删除文件夹，笔记移到"所有笔记"
2. 删除文件夹及其所有笔记（软删除）
3. 取消操作

## 图片功能实现

### 1. Tiptap 图片扩展配置

**使用扩展：**
- `@tiptap/extension-image` - 基础图片支持
- 自定义粘贴处理器 - 拦截剪贴板图片
- 自定义拖拽处理器 - 拦截拖拽的图片文件

**工具栏：**
- 添加"插入图片"按钮
- 点击后打开文件选择器（支持 JPG/PNG/GIF/WebP）

### 2. 图片压缩处理 (electron/utils/image.ts)

**压缩策略：**
- 最大宽度/高度：1920px（超过则等比缩放）
- JPEG 质量：80%
- PNG 转换为 JPEG（除非有透明度）
- 压缩后如果仍大于原图，则保留原图

**实现方式：**
- 使用 Canvas API 进行压缩
- 返回 Base64 格式（data:image/jpeg;base64,...）

### 3. 图片处理流程

```
用户粘贴/拖拽图片
→ 渲染进程接收图片
→ 通过 IPC 发送到主进程压缩
→ 返回压缩后的 Base64
→ 插入到编辑器
```

**IPC 通道：**
- `image:compress` - 压缩图片（输入：Buffer，输出：Base64）

### 4. 编辑器中的图片功能

**交互：**
- 点击图片可调整大小（拖拽边角）
- 图片居中/左对齐/右对齐
- 删除图片（选中后按 Delete 键）

**样式：**
- 图片最大宽度 100%（响应式）
- 图片有圆角和阴影效果
- 选中时显示蓝色边框

## 数据迁移和同步

### 1. 数据库迁移 (electron/db/migrations.ts)

**迁移步骤：**
1. 检测当前数据库版本（存储在 `pragma user_version`）
2. 执行迁移脚本：
   - 创建 `folders` 表
   - 给 `notes` 表添加 `folder_id` 字段（默认值 'all'）
   - 从 localStorage 迁移现有文件夹到数据库
   - 创建索引
3. 迁移完成后更新版本号

**版本管理：**
- 当前版本：1
- 目标版本：2

### 2. 同步协议更新

**新增接口：**
- `GET /api/folders` - 获取所有文件夹
- `POST /api/folders/sync` - 同步文件夹变更

**笔记同步更新：**
- 笔记同步时包含 `folder_id` 字段
- 冲突处理：文件夹冲突按时间戳，后修改的覆盖

### 3. 向后兼容

- 旧版本笔记（没有 folder_id）自动归到"所有笔记"
- 如果移动端不支持文件夹，笔记仍可正常同步（忽略 folder_id）

## 错误处理和边界情况

### 1. 文件夹操作的边界情况

**限制：**
- 不能删除"所有笔记"系统文件夹
- 不能将文件夹移动到自己的子文件夹下（避免循环引用）
- 文件夹名称不能为空，最大长度 100 字符
- 文件夹嵌套最大深度限制为 5 层

**验证：**
- 创建/重命名时验证名称
- 移动时检测循环引用
- 超过深度限制时提示用户

### 2. 笔记移动的错误处理

**异常情况：**
- 目标文件夹不存在时，笔记移到"所有笔记"
- 移动失败时显示错误提示并回滚
- 批量移动时，部分失败不影响其他笔记

**用户反馈：**
- 移动成功显示 toast 提示
- 移动失败显示错误对话框

### 3. 图片处理的错误处理

**限制：**
- 仅支持 JPG/PNG/GIF/WebP 格式
- 图片大小限制：10MB
- 压缩失败时回退到原图（如果原图不超过 5MB）

**错误提示：**
- 格式不支持：提示用户并拒绝插入
- 图片过大：提示用户并拒绝插入
- Base64 解码失败：显示占位图

### 4. 数据完整性

**定期检查：**
- 删除文件夹前检查是否有笔记引用
- 启动时检查孤立笔记（folder_id 指向不存在的文件夹）并修复到"所有笔记"

## 性能优化

### 1. 文件夹树渲染优化

**策略：**
- 使用虚拟滚动（如果文件夹数量超过 100 个）
- 懒加载子文件夹（点击展开时才加载）
- 缓存文件夹树结构，避免重复计算

**实现：**
- 使用 `vue-virtual-scroller` 或自定义虚拟滚动
- 展开状态保存到 localStorage

### 2. 笔记列表优化

**策略：**
- 分页加载笔记（每页 50 条）
- 滚动到底部时自动加载下一页
- 切换文件夹时取消未完成的加载请求

**实现：**
- 使用 Intersection Observer 检测滚动到底部
- 使用 AbortController 取消请求

### 3. 图片优化

**策略：**
- 编辑器中图片懒加载（进入视口才解码 Base64）
- 大笔记（>1MB）延迟加载内容
- 搜索结果不加载图片，只显示文本预览

**实现：**
- 使用 Intersection Observer 实现图片懒加载
- 搜索时使用 `extractTextFromTiptap` 提取纯文本

### 4. 数据库查询优化

**策略：**
- 使用索引加速文件夹和笔记查询
- 笔记数量统计使用缓存（5 秒过期）
- 批量操作使用事务

**实现：**
- 创建必要的索引（已在数据结构设计中定义）
- 使用 Map 缓存笔记数量
- 批量操作使用 `db.transaction()`

## 风险和注意事项

1. **数据迁移风险**: 迁移前需要备份数据库
2. **图片体积**: Base64 会增大数据库体积，需要监控
3. **性能**: 大量图片可能影响编辑器性能，需要懒加载
4. **同步**: 图片数据会增加同步流量，需要考虑网络状况
5. **循环引用**: 文件夹移动时需要严格检查，避免循环引用
