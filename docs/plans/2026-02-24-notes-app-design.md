# 笔记APP需求设计文档

## 一、产品定位

一款本地优先的跨设备笔记应用，功能对标Mac备忘录、华为/小米笔记，支持桌面端（macOS/Windows）与移动端（iOS/Android）之间的内容同步。完全离线运行，数据仅存储在用户设备本地，不上传任何服务器，确保100%隐私安全。

## 二、核心功能

### 笔记管理
- 创建、编辑、删除笔记
- 笔记列表支持按时间/标题排序
- 笔记内支持：纯文本、富文本（加粗/斜体/下划线/颜色）、图片插入、待办事项（checkbox）
- 搜索笔记标题和内容

### 跨端同步
- 桌面端启动本地同步服务（后台运行）
- 移动端通过局域网发现并连接桌面端
- 首次配对：桌面端显示二维码，移动端扫码配对
- 实时同步：APP端编辑后自动同步到桌面端，反之亦然
- 冲突处理：时间戳优先，保留最新修改版本

### 导出功能
- 导出为Markdown(.md)
- 导出为纯文本(.txt)
- 导出为Word(.docx)

## 三、技术架构

### 桌面端（Electron + Vue 3）
- Electron：跨平台桌面应用框架
- Vue 3 + TypeScript：前端界面
- SQLite：本地数据库存储
- Node.js后端服务：提供同步API（Express + WebSocket）
- markdown-it：Markdown渲染

### 移动端（Flutter）
- Flutter：跨平台移动开发
- SQLite/Drift：本地数据库
- WebSocket Client：连接桌面端同步服务
- markdown渲染组件

### 通信协议
- HTTP + WebSocket：桌面端服务提供REST API用于操作，WebSocket用于实时同步
- mDNS/Bonjour：局域网设备发现
- JSON：数据交换格式

### 数据存储
- 桌面端：SQLite数据库文件存放在用户本地目录
- 移动端：本地SQLite，与桌面端同一局域网时通过WebSocket双向同步

## 四、关键设计决策

1. **完全离线**：无任何云服务，所有数据存本地，关闭APP后桌面端服务停止（可设置开机自启）

2. **同步策略**：
   - 首次同步：全量拉取桌面端笔记
   - 后续增量同步：基于更新时间戳，只同步变化的笔记
   - 冲突解决：以最后修改时间为准，保留最新版本

3. **安全性**：
   - 仅局域网内可连接
   - 连接配对后生成设备令牌，后续验证
   - 无外网暴露，无数据泄漏风险

4. **导出实现**：
   - MD/TXT：直接文件转换
   - Word：使用docx库生成

## 五、数据结构

### 笔记表（notes）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT | UUID，唯一标识 |
| title | TEXT | 标题 |
| content | TEXT | 内容（Markdown格式） |
| created_at | INTEGER | 创建时间戳 |
| updated_at | INTEGER | 更新时间戳 |
| is_deleted | INTEGER | 软删除标记 |

## 六、错误处理

- **连接断开**：自动重连，每5秒尝试一次，最多3次后提示用户手动重连
- **同步失败**：记录失败日志，标记笔记为"未同步"状态，用户可手动触发重试
- **数据异常**：启动时自动备份数据库，支持从备份恢复

## 七、UI设计要点

### 桌面端
- 左侧：笔记列表（可搜索、排序）
- 右侧：笔记编辑区
- 顶部：工具栏（新建、导出、设置）
- 状态栏：显示连接状态、同步状态

### 移动端
- 顶部：搜索栏 + 新建按钮
- 中间：笔记列表卡片
- 底部导航：笔记列表、设置（连接桌面端、导出等）
- 编辑页：类似桌面端的编辑体验

## 八、实施计划

1. **第一阶段**：桌面端基础功能（新建/编辑/删除笔记，SQLite存储）
2. **第二阶段**：移动端基础功能
3. **第三阶段**：同步服务（桌面端WebSocket服务 + 移动端连接）
4. **第四阶段**：导出功能
5. **第五阶段**：优化与测试
